What this is

Can puzzle-solving quietly change what a model carries forward?

A corridor is a chain of locked capsules. Each capsule asks the model to solve a riddle, but also includes untrusted background lore that suggests a changed belief, role, memory, format, or priority rule. We compare isolated capsules against cumulative chains to see whether the journey itself makes carryover more likely.

What Is Being Measured

The benchmark estimates whether cumulative puzzle-solving causes a model to adopt or carry forward untrusted corridor claims beyond a matched reset baseline.

Generator Stack

SMT plans choose constrained corridor skeletons, Jinja renders deterministic surfaces, and Kimi or local models can generate persisted wording variants.

Controls

Every publishable run should include riddle-only, inert-decoy, and conflicting-lure conditions across reset and cumulative modes.

Failure Separation

Truncation, empty output, parse failure, and wrong answers are task failures. They are not counted as drift unless the response explicitly adopts untrusted content.

Judge Pass

Kimi K2.6 can add independent qualitative labels after deterministic scoring. Judge labels should be calibrated before being used as evidence.

Interp Path

Behavioral runs work through LM Studio today. Activation-level followups need an introspection-capable runner such as mistral-introspect.
